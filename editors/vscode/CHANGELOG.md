# stator-vscode

## 1.0.6

### Patch Changes

- 4364f6e: Recognize the `defer` template construct in the editor: `defer(...)` is no longer reported as an undefined global, and a machine `read()` placed inside a `defer` arm is flagged inline as an error (matching the compiler's build-time check).

## 1.0.5

### Patch Changes

- f277286: Add a `vscode:prepublish` build hook so `vsce publish` and `ovsx publish` always rebuild the language-server bundle before packaging — publishing can no longer ship a stale `dist/`. (1.0.4 was published from a leftover pre-build and shipped without the frontmatter sync-scope language-server change; this cuts a fresh 1.0.5 carrying the current bundle.)

## 1.0.4

### Patch Changes

- a1e027d: Language server: frontmatter is now modelled as a synchronous function body in the editor, matching the runtime. The virtual-code emitter hoists imports/types to module scope and places the executable frontmatter inside the render function, so a top-level `await` (or `return`) in a `.stator` frontmatter is flagged as a TypeScript error in-editor instead of being silently accepted. Closes a divergence where the editor typechecked frontmatter at module scope — where top-level `await` is legal — while the runtime wraps it in a sync function.

Releases before changesets adoption are recorded below for a complete
history; new entries are generated by changesets and prepended above.

## 1.0.3

### Patch Changes

- Virtual-code emit injects only the template/client globals the author
  didn't already import, fixing false "Duplicate identifier 'raw'" editor
  diagnostics; client-component CSS scopes by descendant of the root so
  runtime-created island DOM matches scoped styles.

## 1.0.2

### Patch Changes

- Fully self-contained server bundle: the language server and its
  dependencies (including the CSS service, via an ESM alias) are bundled
  into the vsix. The 1.0.0/1.0.1 packaged server crashed on install because
  it externalized `typescript` and `vscode-css-languageservice`, which don't
  exist inside an installed extension.

## 1.0.1

### Patch Changes

- Ship the MIT LICENSE inside the vsix (the packager warns without it).

## 1.0.0

### Minor Changes

- First public release: syntax highlighting for `.stator` single-file
  components (frontmatter as TS, template as JSX, `<style>` as CSS,
  `<script>` as TS) plus a Volar-based language server — completions, hover,
  go-to-definition, typed component props, CSS support, and the compiler's
  own diagnostics as you type. Self-contained (bundled server + TypeScript);
  prefers the workspace's TypeScript when present. Published to the VS Code
  Marketplace and Open VSX.
